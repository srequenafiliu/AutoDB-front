En su estado actual, las querys generadas solo son ejecutables en PostgreSQL. No obstante, nuestra visión es que esta aplicación web sea versátil y compatible con cualquier base de datos, buscando así automatizar de manera más integral el proceso.

De momento solo se pueden generar queries con una tabla. Pretendemos ampliar el número de tablas para generar una query más completa.

Una vez que se puedan añadir más tablas en la query, queremos poder relacionarlas entre ellas con claves foráneas.

La query generada puede contener errores, por eso una de nuestras ideas es crear un apartado para validar las queries generadas en AutoDB.

Los usuarios podrían iniciar sesión en AutoDB para consultar todas las queries que ha generado miestras su cuenta estaba activa. De esta manera, estas consultas no se perderían al refrescar el formulario.

Usualmente, los datos dentro de las bases de datos se guardan utilizando formularios en páginas web. Por tanto, queremos implementar otro apartado para generar un formulario que tenga los mismos campos que la query generada.
